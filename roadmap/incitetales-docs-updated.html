What Incitetales is

Northeast India's first AI-powered travel intelligence platform built on verified local knowledge. Core product: an AI itinerary builder anchored in a structured destination database built from personal field experience across Assam, Arunachal Pradesh, Meghalaya, Nagaland, Mizoram, and Sikkim.

Revenue model — in order of activation:

1. Booking commission — Traveller generates an itinerary, selects experiences (stays, guides, activities, transport). Incitetales books everything on their behalf and keeps a small commission per booking. Operators keep the majority. This is the primary revenue engine once operator network is live.

2. Operator monthly subscription — Homestays, taxi operators, local guides, and experience providers pay a fixed monthly subscription to be listed, verified, and featured within Incitetales itineraries. Subscription-only — no per-booking deduction on top. Operators who earn more from Incitetales bookings move to higher tiers.

3. Newsletter sponsorship — The Incitetales weekly newsletter reaches a pre-qualified Northeast India travel audience. Regional tourism brands, hospitality businesses, and travel companies sponsor issues at fixed rates. Editorial independence maintained.

4. NE India knowledge API (Phase 4) — As the verified destination database grows across all eight Northeast states, the structured intelligence becomes licensable to travel companies, OTAs, and state tourism departments who need accurate NE India data they cannot build themselves.

Site Architecture

Incitetales runs on a deliberately lean architecture — static HTML on GitHub Pages, no framework, no build step. Every page is hand-authored, every CSS class is intentional, every line of JS earns its place. The stack was chosen to stay under ₹1,000/month forever, deploy in seconds, and never go down because a dependency broke. The intelligence lives in the data — not the infrastructure.

Matching Logic

// Fuzzy destination match — case-insensitive substring
function destinationMatches(scopeArray, destination) {
  const dest = destination.toLowerCase().trim();
  const tokens = dest.split(/[\s,]+/).filter(t => t.length > 2);

  return scopeArray.some(scope => {
    const s = scope.toLowerCase().trim();
    if (s.includes(dest) || dest.includes(s)) return true;
    return tokens.some(token => s.includes(token) || token.includes(s));
  });
}

// Filter: verified=true only, top 5 by hidden_score

Key Supabase queries to know

-- Verify a story's insights after QA
UPDATE insights SET verified = true WHERE story_slug = 'your-story-slug';

-- Add a correction to verified_facts
UPDATE insights
SET verified_facts = array_append(verified_facts, 'Your correction here')
WHERE story_slug = 'your-story-slug';

-- See all unverified insights
SELECT story_slug, hidden_score, created_at
FROM insights WHERE verified = false ORDER BY created_at DESC;

-- See what InsightEngine would return for a destination
SELECT story_slug, geographic_scope, hidden_score
FROM insights WHERE verified = true ORDER BY hidden_score DESC;
